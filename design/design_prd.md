# ArtRun — Design PRD

> 목적: **Stitch에 넣어 화면 디자인을 뽑기 위한 단일 디자인 브리프.**
> 기술·운영 세부는 전부 제거. 제품의 본질·가치관·화면 플로·비주얼만 담는다.

---

## 1. 한 줄 정의

> **지도 위에 점을 찍거나 손가락으로 긋기만 하면 보행 경로를 따라 러닝 아트 코스가 그려지고, GPX로 내려받아 바로 뛸 수 있는 웹 도구.**

이름: **ArtRun** · 액센트 컬러: **`#BDEFFC`** · 플랫폼: **Web (모바일 우선)**

---

## 2. 왜 만드나 (비즈니스 관점)

러너들 사이에서 GPS 아트런(달리기로 그림 그리기) 열풍이 있다. 고구마런·하트런·고래런처럼 재밌는 코스들이 공유되지만, 정작 **직접 설계해서 뛰어보려면** 구글맵 앞에서 몇 시간을 손그림으로 싸워야 한다. 공유된 코스를 따라 뛰고 싶어도 **그대로 옮길 방법이 없다.**

ArtRun은 이 둘을 해결한다.

1. **클릭/드래그로 설계** — 점 찍거나 쭉 긋기만 하면 인도·보행로를 따라 자동으로 이어진다
2. **원 탭 다운로드** — 완성된 경로를 GPX로 받아 카카오맵·스트라바에 바로 투입
3. **큐레이션 갤러리** — 큐레이션된 코스를 지도 위에 상시 얹어 "발견의 즐거움"까지

**우리가 팔고 싶은 감정**: 도구의 엄격함이 아니라 **놀이의 가벼움**. 계산기가 아니라 스케치북.

---

## 3. 타깃 유저

- **주 사용자**: 주 3회 이상 러닝하는 20~30대. 스트라바에 기록 공유하고, 인스타에서 아트런 사진 보며 "나도 해볼까" 생각한 적 있는 사람.
- **심리적 장벽**: "경로 설계가 너무 번거로워 보여." → ArtRun은 이 장벽을 **"탭 한 번"**으로 낮춘다.

---

## 4. 제품의 5가지 원칙 (비즈니스 가치관)

디자인 판단이 흔들릴 때 이 다섯 개로 되돌아온다.

| # | 원칙 | 뜻 |
|---|---|---|
| 1 | **보행 경로로만 이어진다** | 직선 폴리라인은 최종 결과가 아니다. 계산 중 상태로만 존재한다. |
| 2 | **하나의 맵이 곧 앱** | 페이지 이동 없음. 사이드바·바텀시트·모달로만 모든 걸 해결. |
| 3 | **지도가 탐색 도구** | 카드 리스트를 안 열어도, 지도만 봐도 코스가 옅게 보인다. |
| 4 | **모바일이 정식, 데스크톱은 확장** | 러너는 폰으로 본다. |
| 5 | **놀이 같은 가벼움** | 도구지만 귀엽고, 가볍고, 한 손으로. |

## 5. 비주얼 언어

### 5.1 컬러 (고정, 변경 금지)

| 역할 | HEX | 용도 |
|---|---|---|
| 액센트 | **`#BDEFFC`** | 경로 라인, 활성 상태, primary 버튼. 라이트 아이시 블루. |
| 잉크 | `#0B1220` | 기본 텍스트, 점 마커 테두리 |
| 뮤트 | `#6B7280` | 서브 텍스트 |
| 배경 | `#FFFFFF` | 카드·시트 |
| 라인 | `#E5E7EB` | 구분선 |
| 위험 | `#EF4444` | 삭제 확인 |

> `#BDEFFC`는 면적으로 쓰지 말 것. **액센트**로만. 경로와 활성 상태에만 등장.
> 지도 위 경로 라인은 **흰 halo 2px + `#BDEFFC` 안쪽**의 2레이어 스택으로 (대비 보강).

### 5.2 타이포그래피

- 폰트: **Pretendard** (한국어 친화 시스템 폰트 스택)
- 스케일:
  - `display` 28/700 — 모달 타이틀
  - `title` 20/700 — 탭 헤더
  - `body-lg` 16/600 — 카드 이름, 버튼
  - `body` 14/500 — 본문
  - `caption` 13/500 — 서브정보

### 5.3 공간 & 형태

- 간격: `4 / 8 / 12 / 16 / 20 / 24 / 32`
- 라운드: 버튼 8, 카드 12, 시트 16, 모달 16
- 카드 그림자: `0 2px 12px rgba(0,0,0,0.08)` — 은은하게
- 아이콘: Lucide outline 2px, 모두 `currentColor`

### 5.4 무드 (한 문장씩)

- **친근하고 가볍게.** 존댓말인데 어깨에 힘 뺌. "~하세요" 대신 "~해요".
- **과하지 않은 모션.** 150~250ms, ease-out 기본. 도구의 반응성 = 품질.
- **귀여움 vs 진지함 = 70:30.** 러닝 커뮤니티 정서(도전·성취·웃음) 존중.

## 7. 화면 플로 (핵심 3개)

### A. "갤러리에서 골라서 바로 뛰기" (가장 흔한 경로)

```
지도 열림 (코스들이 옅게 얹혀 있음)
  ↓
사이드바/바텀시트 열기
  ↓
탭 "인기 아트런" 선택
  ↓
"여의도 고구마런" 카드 탭
  ↓
지도가 여의도로 부드럽게 이동 + 해당 경로 강조
  ↓
[GPX 저장] 탭 → 다이얼로그 → 저장
  ↓
끝. (카카오맵 등에 넣으러 앱 밖으로)
```

### B. "처음부터 점으로 그리기"

```
지도 탭
  ↓
자동으로 그리기 모드 진입 + 점 1개 생김
  ↓
계속 탭 → 인도 따라 선 자동 연결 (연결 중 회색 점선 → 완료되면 #BDEFFC 선)
  ↓
점 드래그/삭제로 미세 조정
  ↓
총거리 확인 → [GPX 저장]
```

### C. "손가락으로 쭉 그리기"

```
모드 토글 → [드래그] 선택
  ↓
지도 위에서 손가락으로 쭉 그림
  ↓
그리는 동안 포인터 궤적이 연한 점선 (계산 전이라는 신호)
  ↓
손 뗌 → "경로 계산 중…" 토스트
  ↓
세그먼트가 앞에서부터 순차적으로 #BDEFFC 선으로 채워짐
  ↓
[GPX 저장]
```

## 8. Stitch로 생성할 화면 (우선순위 순)

> 각 화면은 독립적으로 Stitch에 프롬프트로 넣을 수 있도록 "씬 묘사" 형태로 작성.

### #1 — 메인 / 탐색 중 (모바일) ★★★

지도가 화면 전체를 채움. 서울 중심 정도의 줌. 지도 위에는 큐레이션된 코스 6~8개가 **`#BDEFFC` 반투명(35%) 라인**으로 옅게 얹혀 있음. 인기 코스는 실선, 인기 아트런은 대시 패턴(`[2,2]`).

상단에 얇은 타이틀 바 `ArtRun` (좌측) + 메뉴 아이콘(우측).

화면 우측에 세로로 **플로팅 툴바** — 흰 카드, 은은한 그림자, 버튼 6개 세로 나열 (↶ ↷ 🗑️ ⛶, 구분선, 거리 라벨 `0.0 km`, primary 버튼 `GPX ↓`). 경로 없는 상태라 대부분 비활성(회색 톤).

화면 하단 중앙에 **모드 토글** — `점 / 드래그 / 선택` 세그먼트 컨트롤, 기본은 `점` 활성.

화면 맨 아래 **바텀시트 peek** (80px) — 얇은 드래그 핸들 + "인기 코스 · 인기 아트런" 탭 제목만 살짝 보임.

지도 빈 곳에 작은 힌트 고스트 텍스트: "지도를 탭해서 그림 그리기를 시작해요"

### #2 — 메인 / 탐색 중 (데스크톱) ★★★

좌측 320px **사이드바**. 상단에 워드마크 `ArtRun`. 아래 탭 2개: `인기 코스 (6)` `인기 아트런 (8)`. 기본은 `인기 아트런` 선택.

탭 아래로 카드 스크롤 리스트. 각 카드:
- 썸네일 120×80 (webp 분위기의 지도 그림) radius 8
- 제목 "여의도 고구마런" 16/700
- 서브 "8.0km · 서울 영등포" 13/500 뮤트
- 설명 2줄 "한강을 따라 고구마 모양 완성" 뮤트
- 하단 우측에 작은 버튼 2개: `[편집하기]` (ghost) `[GPX 저장]` (filled `#BDEFFC`)

우측 영역은 지도 전체 (모바일과 동일한 오버레이 + 툴바). 툴바 위치는 우상단.

### #3 — 갤러리 바텀시트 확장 (모바일) ★★

모바일 메인과 동일하되, 바텀시트가 **half 45vh**로 확장된 상태. 탭 2개 (`인기 코스` `인기 아트런`)가 크게 보임. 탭 아래 카드 2~3장이 세로로 나열. 카드 구조는 데스크톱과 동일하지만 썸네일이 100×100 정사각.

지도 영역이 상단 55vh로 줄어든 상태.

### #4 — 그리기 중 / 점 모드 (모바일) ★★★

지도 중앙에 5~7개의 점이 찍혀 있음. 각 점은 **흰색 원 (radius 10) + `#BDEFFC` 원(radius 7) + 테두리 `#0B1220` 1px**, 가운데에 번호(1~N).

점들 사이는 **흰 halo 2px(바깥) + `#BDEFFC` width 5(안쪽)**의 굵은 선. 마지막 세그먼트 하나만 회색 점선 (계산 중 상태).

툴바가 활성 상태 — 버튼들 진하게, 거리 라벨 `2.8 km` 또렷하게, `GPX ↓` 버튼이 primary로 강조.

모드 토글은 `점` 활성.

### #5 — 그리기 중 / 드래그 모드 (모바일) ★★

모드 토글이 `드래그` 활성. 지도 위에 유저의 포인터 궤적이 연한 점선 자유곡선으로 그려져 있음 (아직 라우팅 전). 궤적 따라 30m마다 점이 있을 듯한 위치에 미세한 흐린 점 마커.

상단 토스트: "경로 계산 중…" (은은한 흰 카드, info 아이콘)

하반부 세그먼트는 이미 `#BDEFFC` 선으로 바뀌어 progressive하게 채워지는 중 (상반부는 아직 점선).

### #6 — GPX 저장 다이얼로그 ★★

화면 전체에 dim 50% 오버레이. 중앙 모달 (radius 16, 흰 배경, shadow 강하게).

구성:
- 타이틀 "GPX 저장하기" (display 28/700)
- 입력 필드: placeholder `내 아트런`, label "코스 이름" 상단
- 토글 한 줄: `타임스탬프 포함` (ON 상태). 아래 설명 caption "Strava에서 route로 인식되려면 필요해요"
- 버튼 행: `[취소]` (ghost) · `[저장]` (primary `#BDEFFC`, 오른쪽 정렬)

### #7 — Clear 확인 다이얼로그 ★

화면 dim. 중앙에 작은 모달.
- 타이틀 "모든 점을 지울까요?"
- 설명 "되돌릴 수 있지만, 한 번에 모두 사라져요."
- 버튼: `[취소]` `[다 지우기]` (danger, `#EF4444`)

## 9. 컴포넌트 카탈로그 (Stitch 참고용)

| 컴포넌트 | 특징 |
|---|---|
| **Primary 버튼** | filled `#BDEFFC`, 잉크 텍스트, radius 8, height 44, hover 살짝 진해짐 |
| **Secondary 버튼** | 흰 배경 + `#E5E7EB` 테두리, 잉크 텍스트 |
| **Ghost 버튼** | 배경 없음, 잉크 텍스트, hover 시 라이트 그레이 배경 |
| **Danger 버튼** | `#EF4444` fill, 흰 텍스트 |
| **세그먼트 컨트롤** | 3분할, 활성 세그먼트 `#BDEFFC` 배경 + 슬라이딩 인디케이터 |
| **탭** | 언더라인 스타일, 활성 탭 밑에 `#BDEFFC` 두께 2px 바 |
| **카드** | radius 12, 흰 배경, shadow 은은하게, 내부 패딩 16 |
| **토스트** | 흰 카드, 라운드 8, 아이콘 + 메시지, 상단 중앙(모바일)/우하단(데스크톱) |
| **바텀시트** | 상단 라운드 16, 드래그 핸들 36×4, 3단 스냅 |
| **모달** | 중앙 정렬, radius 16, dim 50% 오버레이 |
| **지도 점 마커** | 흰 halo + `#BDEFFC` + 잉크 테두리 + 번호 라벨 |
| **지도 경로 라인** | 흰 halo 2px + `#BDEFFC` 5px (완료) / 회색 점선 (계산 중) |

## 10. 카피 톤 (짧게)

- 존댓말. 어깨 힘 뺌. "~해요" 스타일.
- CTA는 짧고 능동적으로: `GPX 저장`, `편집하기`, `다 지우기`
- 이모지는 안 씀 (인터페이스에서). 단, 지도 심볼이나 토스트의 아이콘은 아이콘 컴포넌트.
- 주요 문구:
  - 힌트: "지도를 탭해서 그림 그리기를 시작해요"
  - 로딩: "경로 계산 중…"
  - 성공: "artrun_0417.gpx 저장됨"
  - 경고 (드래그 상한): "경로가 충분히 길어요. 손을 떼면 계산을 시작할게요."

## 12. 한 장 요약 (Stitch 프롬프트 용)

> **ArtRun은 지도 한 화면 위에서 점을 찍거나 손가락으로 쭉 그어서 보행 경로를 따라 러닝 코스를 설계하고 GPX로 내려받는 웹앱이다. 모바일 우선. 액센트 컬러 `#BDEFFC` 한 톤. 큐레이션된 인기 코스와 인기 아트런이 지도 위에 상시 옅게 얹혀 있고, 사이드바(데스크톱) 또는 바텀시트(모바일)의 탭 2개로도 탐색 가능하다. 지도 위 플로팅 툴바에서 Undo/Redo/Clear/Fit/GPX 저장을 한다. 모드 토글로 점 모드와 드래그 모드를 전환. 느낌은 도구의 엄격함이 아니라 스케치북의 가벼움 — 가볍고, 귀엽고, 한 손으로.**
